# Reflexion final

## Beneficios

- Acelera la planificacion de funcionalidades.
- Ayuda a explicar conceptos como LocalStorage, accesibilidad y filtrado.
- Propone alternativas cuando hay varias formas de resolver algo.
- Facilita documentar el proceso y no solo el resultado final.

## Riesgos

- Puede generar codigo que parece correcto pero no encaja con el proyecto.
- Puede olvidar detalles de accesibilidad o persistencia si no se le piden.
- Puede tocar demasiados archivos si el prompt no limita el alcance.
- Puede dar una falsa sensacion de seguridad si no se prueba el codigo.

## Como lo he usado de forma responsable

1. He pedido cambios concretos.
2. He mantenido el stack original.
3. He revisado el diff antes de subir.
4. He ejecutado comprobaciones de sintaxis.
5. He documentado los prompts y conclusiones.

## Copilot frente a Cursor

GitHub Copilot es muy practico para autocompletar codigo mientras se escribe. Cursor es mas potente cuando se necesita hablar con el proyecto completo o modificar varios archivos a la vez.

Mi conclusion:

- Copilot encaja mejor como ayuda continua mientras programas.
- Cursor encaja mejor para refactorizaciones, explicaciones y cambios coordinados.
- ChatGPT o Claude son utiles para planificar, comparar enfoques y documentar.

## Conclusion personal

La IA no hace que el desarrollador deje de pensar. Lo que hace es reducir friccion: ayuda a empezar, comparar ideas y revisar decisiones. La parte importante sigue siendo entender el codigo, probarlo y decidir que cambios tienen sentido.
